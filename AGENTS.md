# AGENTS.md

## 프로젝트 개요

이 프로젝트는 실제 입양을 대신하지 않고, 강아지 양육에 필요한 반복적인
돌봄과 시간 부담을 미리 체험하게 하는 웹 서비스입니다.

## 디렉터리 구조

```text
src/
├── index.html                 # 화면 구조와 접근성 속성
├── app.js                    # 도메인 로직, 저장, 브라우저 UI 연결
├── styles.css                # 화면 스타일과 반응형 레이아웃
└── data/care-scenarios.json  # 돌봄 유형과 시뮬레이션 시나리오

server/index.js               # 정적 파일 서버
tests/simulation.test.js      # Node 내장 테스트
tests/acceptance.md           # 브라우저 수동 검증 항목
specs/001-puppy-care-simulation/
                              # 명세·계획·계약·작업 목록
```

## 실행과 검증

```bash
npm start   # http://localhost:3000
npm test    # Node 내장 테스트 실행
```

기능을 바꾸면 `npm test`를 먼저 실행하고, 변경된 사용자 흐름은
`tests/acceptance.md`에도 반영합니다. 새 라이브러리는 기존 Node 런타임과
브라우저 API로 해결할 수 없는 경우에만 추가합니다.

## 코드 작성 규칙

### 함수 분리

- 함수는 하나의 책임만 가집니다. 생성, 상태 전이, 저장, 요약 계산, 렌더링을
  한 함수에 섞지 않습니다.
- `src/app.js`의 도메인 함수는 브라우저 전역(`document`, `localStorage`)에
  의존하지 않게 유지하고, Node 테스트에서 직접 import할 수 있게 합니다.
- 상태 변경은 유효한 전이만 허용하고, 완료 처리는 멱등적으로 유지합니다.
  같은 요청을 두 번 완료해도 기록과 집계가 중복되면 안 됩니다.
- 저장 실패 시 기존 상태를 유지하고 사용자가 재시도할 수 있는 오류를
  반환합니다.
- UI 함수는 상태 계산을 새로 구현하지 말고 도메인 함수의 결과를 렌더링합니다.
- 서버 코드는 정적 파일 제공만 담당합니다. 사용자 세션이나 돌봄 기록을
  서버로 보내는 기능을 추가하지 않습니다.

### 한국어 주석

- 주석은 한국어로 작성합니다.
- 코드가 무엇을 하는지 반복 설명하지 말고, 왜 그런 선택을 했는지 설명합니다.
- 알림 권한 거부 시 앱 내 목록을 기준 상태로 삼는 등, 브라우저 제약이나
  제품 정책처럼 코드만 읽어서는 알기 어려운 이유를 우선 주석으로 남깁니다.
- 주석으로 구현을 대신하지 않습니다. 상태 전이·검증 규칙은 코드와 테스트로
  표현하고, 주석은 보충 설명에만 사용합니다.

### 사용자 경험과 안전

- 시스템 알림은 선택 사항입니다. 알림을 거부하거나 놓쳐도 앱 안에서 요청을
  확인할 수 있어야 합니다.
- 키보드 조작, visible focus, 접근 가능한 이름, `aria-live` 상태 안내를
  유지합니다.
- 시뮬레이션 결과로 입양 적합성이나 준비도를 판정하지 않습니다.
- 매칭, 분양, 입양 신청, 판매 기능을 추가하지 않습니다.
- 개인 생활 패턴은 최소한만 저장하고, 새로운 저장 데이터가 필요하면 목적과
  보관 범위를 명세에 먼저 반영합니다.

## 변경 작업 순서

1. 관련 명세와 계약을 확인합니다.
2. 도메인 함수와 테스트를 먼저 수정합니다.
3. UI와 스타일을 연결합니다.
4. `npm test`와 관련 acceptance check를 실행합니다.
5. 작업 목록(`specs/001-puppy-care-simulation/tasks.md`)을 실제 상태와 맞춥니다.

작은 변경은 기존 구조를 재사용합니다. 한 곳에서만 쓰이는 추상화, 새 상태
관리 계층, 별도 서비스 계층은 실제 필요가 생길 때까지 만들지 않습니다.
